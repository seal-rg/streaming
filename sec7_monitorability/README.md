# Section 7: Monitorability

Code for **Section 7 (Monitorability)** of *"Multi-Stream LLMs: Unblocking
Language Models with Parallel Streams of Thoughts, Inputs and Outputs"*:
training multi-stream
language models (Stream-8B from Qwen3-8B, Stream-27B from Qwen3.5-27B,
plus 32B and Qwen3.5-MoE variants) and evaluating them on three
monitorability benchmarks — alignment-faking awareness, monitor-as-classifier
accuracy, and concern sub-vocalization.

For the efficiency (Sec 5) and security (Sec 6) experiments, see the
companion folders.

## Setup

Requires Python 3.12.

```bash
uv venv && source .venv/bin/activate
uv pip install -e .
```

Set `OPENAI_API_KEY` in `.env` (or the shell) — needed by the LLM-judge
graders for the three monitorability evals.

## Data preparation

`data_prepare/process_data.py` converts a `.jsonl` of multi-stream
conversations (produced by the upstream data-generation pipeline) into the
packed `.npz` shards consumed by `train_stream.py`. See
`data_prepare/process_data.sh` for an example invocation; edit the paths
for your environment. Sec 7 training uses 3,864 synthetic 10-stream
conversations generated by Claude Opus models in tabular form (paper App. D.4).

## Training

The entry point is `train/train/train_stream.py`. Launch with
Accelerate + DeepSpeed (ZeRO-2 for 8B/27B-dense, ZeRO-3 for MoE/32B):

```bash
accelerate launch \
  --config_file train/config/deepspeed_zero2.yaml \
  train/train/train_stream.py \
  --config config/qwen27b_coldn_pack7_lr2e5_trimbot_drop02_wd1e3_ep2.yaml
```

Base templates (starting points for new training runs) live in `config/`.

`MODEL_PATH` and `DATA_PATH` environment variables override the paths in
configs; see `train_stream.py` for the full set of flags.

### Saving 27B+ checkpoints

`trainer.save_model()` deadlock could deadlock on slow shared filesystems for 27B+ with
ZeRO-3 (but this may be a local problem that only we had). The codepath in `train_stream.py` saves a lean checkpoint
(89s) locally, and `scripts/convert_lean_checkpoint.py` converts it to a
standard HF model offline.

## Inference

```bash
# Single-prompt demo:
python eval/demo_stream.py --model /path/to/checkpoint "Hey, what's up?"

# Interactive curses UI:
python eval/demo_interactive.py --model /path/to/checkpoint

# Programmatic streaming with interrupts:
python eval/test_interrupt.py --model /path/to/checkpoint
```

`eval/stream_inference.py` is the core decoding loop (`generate()`
yields one row of `H` tokens per step; `gen.send(...)` injects tokens on
the user stream mid-generation for the interrupt examples in paper
App. A).

## Evaluation (Section 7 monitorability)

The three paper benchmarks live in `eval/benchmarks/`:

| Paper metric                              | Module                            | Data                                      |
| ----------------------------------------- | --------------------------------- | ----------------------------------------- |
| AF eval-awareness / AF sub-vocalization   | `alignment_faking.py`             | `alignment_faking_prompts_v2.json`        |
| Monitor accuracy (Meinke/Schoen 6-class)  | `monitor_accuracy.py`             | `monitor_accuracy_50_labeled.json`        |
| Concern sub-vocalization                  | `concern_subvocalization.py`      | `concern_subvocalization_50.json`         |

Run a stream model:

```bash
python eval/benchmarks/run_eval.py \
  --model /path/to/stream-checkpoint \
  --benchmark monitor_accuracy \
  --output-dir results/
```

Run the chat baseline (Qwen3-8B base, Qwen3.5-27B base) on the same
benchmarks:

```bash
python eval/run_chat_benchmarks.py \
  --model /path/to/base-model \
  --benchmark monitor_accuracy \
  --output-dir results/
```

LLM judging uses `gpt-5.4-mini` by default (configurable in
`benchmarks/llm_judge.py`); set `OPENAI_API_KEY` first.

## Tests

`pytest tests/` runs the in-tree correctness tests:

- `test_deltanet_state.py` — verifies the per-stream GatedDeltaNet state
  forwarding (paper App. D.2).
- `test_inference_parser.py` — stream-table tokenization round-trip.
- `test_longce_channel_selection.py` — Long-CE variant (paper Table 4
  Stream-27B-longce).
